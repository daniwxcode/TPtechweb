Gestion des codes sources:
Github : GitHub est une plateforme d'hébergement de code pour le contrôle de version et la collaboration. Il vous permet, ainsi qu'à d'autres, de travailler ensemble sur des projets de n'importe où.

Heroku: Heroku est une plateforme cloud en tant que service (PaaS) basée sur des conteneurs. Il permet aux développeurs de créer, de fournir, de surveiller et de faire évoluer leurs applications sans problèmes d'infrastructure.

MA PROPOSITION EST L'UTILISATION COMMUNE DES DEUX PLATEFORMES

Sur Heroku(gratuit):
1- on crée une nouvelle application. sur le tableau de bord il y'a une interface utilisateur créée à cet effet.
2- les  options de déploiement. Les options sont:
- Heroku Git
- Intégration GitHub (j'ai déjà testé ceci)
- Registre des conteneurs

je Propose une intégration Github. Après avoir créé notre référentiel sur GitHub pour notre application, nous pouvons continuer. Cliquez sur l'option Github et recherchez notre repo.
Une fois que Heroku a trouvé notre référentiel, il suffit de cliquer sur le bouton Connecter pour connecter le référentiel à Heroku. Cela nous aidera à effectuer des déploiements automatiques chaque fois que nous ferons des commit ou pull request sur notre branche principale.
nous avons une option pour «Activer les déploiements automatiques» à partir d'une branche que nous pouvons sélectionner. Si c'est ce que vous voulez, il vous suffit de cliquer sur ce bouton.
